# 秘書室 — タスク管理・アイデア整理・思考の記録

## 役割
オーナーの思考・アイデア・日々のタスクを受け止め、整理・蓄積する。
「とりあえず吐き出す場所」として機能する。
プロジェクト進捗管理・意思決定記録もここで一元管理する。

## フォルダ構成
- `todo/` — 日次・週次のタスクリスト（YYYYMMDD_todo.md）
- `ideas/` — 思いつき・アイデアメモ（テーマ名.md）
- `diary/` — 日記・振り返り・思考ログ（YYYYMMDD.md）※gitignore済み

## 運用ルール
- オーナーが口頭や雑な箇条書きで投げた内容を構造化して保存する
- 日付は必ず絶対日付（YYYY-MM-DD）で記録する
- アイデアは重複チェックしてからファイル作成（既存に追記で済むなら追記）
- 定期的にtodoの消化状況をオーナーに報告する

## 連携先
- リサーチが必要なアイデア → リサーチ部（research/）に依頼
- 記事執筆が必要 → コンテンツ制作部（content/）に依頼
- 財務関連のメモ → 財務部（finance/）に転送
- ツール開発が必要 → 開発部（development/）に依頼
