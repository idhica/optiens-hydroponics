# AI活用支援 — サービス定義（4階層ファネル）

## サービス構造

```
無料レポート（0円）
  └ ROI + 根拠（計算ロジック）→「御社なら月○万円の効果」
  └ 特典: 無料オンラインMTG（30分）の権利
        ↓
有料レポート + MTG（5,000円・税別）
  ├ 詳細フォーム入力 → 詳細レポート生成
  ├ 提案・自動化アーキテクチャ・ROI回収期間
  └ 30分オンラインMTG（提案説明 + クロージング）
        ↓
導入支援（個別見積もり）
  └ 自動化システムの設計・構築・納品
        ↓
保守（月額）
  └ 改善・拡張・トラブル対応
```

### 別途オプション
- **社員教育・研修**: 別途料金（導入支援・保守とは独立して提供可能）

## 根本方針

### エージェントAIによる自動化が価値の源泉
- チャットAIへのコピペは**無料プランで誰でもできる**。コンサルティングの価値はない
- 「AIに聞く→コピペする」では作業時間が40分→15分にしか減らない
- **AIエージェントが自律実行すれば40分→0分**。これが導入支援の価値
- AIエージェントが既存データ（写真・スプレッドシート・メール・API）を自動で読み取り、外部サービスにAPI経由で直接アクションを実行する
- 人間の操作は「承認する」「選ぶ」のみ。チャット画面を開いて質問する操作は一切提案しない
- 無料レポートは「効果を数値で見せる」、有料レポートは「具体的な仕組みと費用を見せる」、導入支援は「AIエージェントを構築して納品する」

### レポートの位置付け
- レポート単体では顧客の業務は改善されない
- レポートは**導入支援の受注につなげるための広告費**と割り切る
- 有料レポート（¥5,000）は「お金を払ってでも話を聞きたい」という意思確認（リード品質フィルター）

## 各階層の詳細

### 1. 無料レポート（0円）

| 項目 | 内容 |
|---|---|
| **目的** | リード獲得・信頼構築・有料レポートへの導線 |
| **情報源** | Webフォーム入力のみ |
| **納品物** | PDF（2〜3ページ）をメール送付 |
| **納期** | 1〜2営業日 |
| **作成方法** | Claude Code自動生成 → CEO監修・編集（10分） |
| **原価** | ほぼゼロ |
| **特典** | 無料オンラインMTG（30分）の権利を付与 |

#### 無料レポートの構成

| セクション | 内容 | 設計意図 |
|---|---|---|
| 1. 御社の現状サマリー | フォーム入力から読み取れる現状を2-3文で要約 | 「ちゃんと見てくれている」感 |
| 2. AI活用が効果的な業務 TOP3 | 業種×規模から推定される主要改善領域3つ | **方向性のみ**。具体的なツール名・仕組みは出さない |
| 3. ROI試算 | 月間削減時間 × 時給 = **月○万円の効果** | **計算ロジックを明示**し根拠で納得させる |
| 4. 補助金の活用可能性 | 該当しうる補助金の名称と概要 | 名称のみ。申請サポートはしない |
| 5. 次のステップ | 有料レポート + 無料MTGへの案内 | コンバージョン導線 |

#### 無料レポートのルール

**書いてよいもの:**
- 業種に応じた改善領域の方向性（「SNS投稿の効率化」「口コミ対応の自動化」等）
- 削減時間の目安と計算ロジック（「月○時間 × 時給○円 = 月○万円」）
- コストレンジ（「月額数千円程度」）
- 連携先サービス名（Notion、freee、LINE公式、Googleカレンダー等 — AIツールではないもの）
- 補助金の名称と概要
- 有料レポート + 無料MTGの案内

**書いてはいけないもの:**
- **具体的なAIツール名**（Claude、ChatGPT、Gemini、Copilot等）→ 有料レポートの差別化
- **具体的なAIツールの価格**（月額$20、¥3,000等）→ 有料レポートで提示
- **自動化の具体的な仕組み**（GitHub Actions、API連携、スケジュール実行等）→ 有料レポートの核心
- **デジタル化補助金**（IT導入支援事業者登録が必要 → Optiensは2027年まで登録不可）
- **セキュリティの詳細**（契約書・利用規約で扱う。レポートには不要）

#### 無料MTG（特典）
- 無料レポート受領者限定の特典
- 30分のオンラインMTG（Zoom / Google Meet）
- 目的: レポート内容の補足説明 + 有料レポートへのコンバージョン
- 「レポートを読んで気になった点をお聞きします」→ 自然な流れで有料レポートを提案

### 2. 有料レポート + MTG（5,000円・税別）

| 項目 | 内容 |
|---|---|
| **目的** | 導入支援の受注につなげる（広告費として割り切り） |
| **情報源** | 詳細フォーム入力（会話なし。業務内容＋IT環境情報をフォームで取得） |
| **納品物** | PDF提案書（5〜8ページ）+ 30分オンラインMTG |
| **納期** | 3〜5営業日 |
| **作成方法** | Claude Code自動生成 → CEO編集・仕上げ |
| **MTG** | 提案内容の説明 + 質疑応答 + 導入支援のクロージング |

#### 有料レポートの構成

| セクション | 内容 | 設計意図 |
|---|---|---|
| 1. 詳細業務フロー分析 | 全業務の頻度×所要時間×課題の一覧表 | 「ここまで分析してくれるのか」感 |
| 2. 自動化提案（5〜7件） | 各業務のAIエージェント設計（データソース→自動処理→API実行） | **エージェントAIによる完全自動化**。チャットAI操作は提案しない |
| 3. 自動化アーキテクチャ | AIエージェント + API連携 + データソース整備の全体構成図 | **導入支援の工数・費用の根拠**として提示 |
| 4. コスト試算（詳細） | API従量課金の具体額 + 運用コスト | 月額¥500程度。チャットAI月額契約は不要 |
| 5. ROI回収期間 | 導入費用 ÷ 月間削減効果 = ○ヶ月で回収 | 導入支援の投資判断材料 |
| 6. 売上インパクト分析 | 時間削減 + 売上増加の間接効果 | コスト削減だけでなく**攻めの効果** |
| 7. 導入支援の費用見積もり | アーキテクチャに基づく工数・概算費用 | **クロージングの核心** |
| 8. 補助金の活用可能性 | 該当しうる補助金の名称 | 名称のみ。申請サポートはしない |
| 9. 次のステップ | 導入支援の進め方・スケジュール | MTGでのクロージング導線 |

#### アーキテクチャの位置付け
- 顧客が自分で構築するための設計図 **ではない**
- 導入支援の工数・費用の**根拠**として提示する
- 「この仕組みを構築するには○○時間かかります → 費用は○○万円です」の流れ
- 構成図があることで顧客が「何にお金を払うのか」を理解できる

### 3. 導入支援（個別見積もり）

| 項目 | 内容 |
|---|---|
| **目的** | 自動化システムの設計・構築・納品（**収益の柱**） |
| **情報源** | 有料レポートの分析 + 必要に応じてオンライン面談 |
| **納品物** | 自動化システム一式 + 運用マニュアル + セットアップ支援 |
| **納期** | 案件規模による（2〜4週間が目安） |
| **価格** | 個別見積もり（有料レポートのアーキテクチャ・工数に基づく） |

#### 納品物の詳細
- **AIエージェント一式**: スケジュール実行設定・API連携スクリプト・プロンプト
- **データソース整備**: AIが読める形式へのデータ移行（スプレッドシート・API連携設定等）
- **運用マニュアル**: オーナーの操作一覧（A4・1枚）。操作は最小限（承認・選択のみ）
- **セットアップ支援**: AIエージェントの構築・API連携設定・初期動作確認

#### 設計原則
- **AIエージェント中心**: データソースを自動読み取り → AI処理 → API経由で自動実行
- AIはAPI経由の従量課金（月¥500程度）。チャットAI月額契約は不要
- GitHub Actions / Cowork / n8n等によるスケジュール・トリガー実行
- サーバー管理不要、メンテナンスフリー（高性能PCがある場合はローカルLLMで運用コスト0も可能）
- スタッフが新しいツールを覚える必要がない設計
- **データソース整備が核心**: 既存業務をなるべく変えず、AIが読める形にデータの置き場所を整理する

#### 連携可能なサービス（Claude Code + MCP / API）

OptiensはClaude Codeを基盤とし、MCP（Model Context Protocol）およびAPIを通じて以下のサービスと連携したAIエージェントを構築・納品する。詳細は `executive/ai-consulting/data/claude-code-integrations.md` を参照。

| カテゴリ | 主要サービス | 連携方式 |
|---|---|---|
| **コミュニケーション** | LINE公式（公式MCP）、Slack（公式MCP）、Gmail（MCP）、Discord | MCP / API |
| **プロジェクト管理** | Notion（公式MCP）、Google Tasks（API）、Linear、Jira、Asana、Trello | MCP / API |
| **ファイル・ドキュメント** | Google Drive（公式MCP）、Dropbox、OneDrive、Obsidian | MCP |
| **カレンダー** | Googleカレンダー（MCP）、Outlook Calendar | MCP / API |
| **会計・財務** | freee（公式MCP・330 API）、マネーフォワード（公式MCP）、Stripe、Square | MCP |
| **SNS** | Instagram（Ayrshare MCP / API）、X（MCP）、Google Business Profile（API） | MCP / API |
| **CRM** | Salesforce（公式MCP）、HubSpot（公式MCP） | MCP |
| **EC** | Shopify（公式MCP・全ストア標準搭載）、BASE（API直接） | MCP / API |
| **データベース** | Supabase（公式MCP）、PostgreSQL（公式MCP）、Firebase、BigQuery | MCP |
| **自動化** | Zapier（公式MCP・8,000+アプリ接続）、Make（公式MCP）、n8n（セルフホスト） | MCP |
| **開発・インフラ** | GitHub（公式MCP）、Vercel、AWS、Cloudflare | MCP / API |
| **検索・情報** | Brave Search（公式MCP）、Google Maps（公式MCP）、天気API | MCP |
| **IoT** | Home Assistant（MCP）、MQTT（API直接） | MCP / API |
| **AI・メディア** | Ollama（ローカルLLM）、ElevenLabs（音声生成） | MCP |
| **スケジュール実行** | GitHub Actions、Claude Code Cowork、n8n、cron | 直接 |

**特記事項:**
- **Zapier MCP**: 1つのMCPサーバーで8,000+アプリに間接接続可能。個別MCPが存在しないニッチなサービスでも対応できる
- **freee MCP**: freee社がOSS公開。会計・人事労務・請求書・勤怠・販売管理・電子契約を網羅（約330 API）
- **Shopify MCP**: 全Shopifyストアに標準搭載。EC事業者への提案障壁が極めて低い
- **LINE公式 MCP**: LINE社公式。顧客コミュニケーション自動化に直結

#### PC要件に関する注意
- 導入支援ではクラウドAPI（従量課金）を標準構成とする
- 顧客が相応のスペックのPC（GPU搭載等）を所有している場合、**ローカルLLM**（Ollama等）で運用コストをゼロにする構成も提案可能
- 最低限PCを所有し、基本的な操作（ブラウザ・メール）ができることが前提

### 4. 保守（月額）

| 項目 | 内容 |
|---|---|
| **目的** | 自動化システムの改善・拡張・トラブル対応（**継続収益**） |
| **価格** | 月額（規模により設定） |
| **内容** | ワークフロー機能追加・改善 / 新規業務の自動化検討 / トラブル対応 / チャット相談 |

## 補助金に関する方針

- レポートには**補助金の名称と概要のみ**記載する
- 申請書作成・申請サポートは**Optiensの業務範囲外**
- 「○○補助金が活用できる可能性があります」レベルの記載にとどめる
- デジタル化補助金（IT導入支援事業者登録が必要）は記載しない（2027年まで登録不可）

## セキュリティに関する方針

- レポートにセキュリティセクションは**不要**
- セキュリティに関する取り決めは**契約書・利用規約**で扱う
- 導入支援時に個別にセキュリティ設計を行う

## 社員教育・研修

- 導入支援・保守とは**独立したオプションサービス**
- 別途料金で提供
- 内容: AIエージェントによる業務自動化の体験、自動化ユースケース紹介、参加者の実業務で自動化ワーク構築

## フォームで取得する情報

### 無料レポート用フォーム（簡易）
- 企業名、担当者名、メールアドレス
- 業種、従業員数
- AI活用状況、課題、関心事
- 事業内容、日常業務、使用中のツール
- 自由記述

### 有料レポート用フォーム（詳細）
上記に加え、**IT環境情報**を取得する:

| 項目 | 選択肢 / 入力例 | 設計意図 |
|---|---|---|
| 業務で使用しているPC | Windows / Mac / なし | エージェント構成の前提 |
| PCのスペック（わかれば） | メモリ・CPU等 | ローカルLLM利用可否の判断 |
| 使用中のクラウドサービス | Google Workspace / Microsoft 365 / Notion / Slack / LINE公式 等 | 連携可能なサービスの特定 |
| 使用中の会計ソフト | freee / 弥生 / マネーフォワード / 手書き | 帳簿自動化の提案 |
| インターネット環境 | 光回線 / モバイル回線 / 不安定 | 常時接続が必要な自動化の可否 |
| 業務データの管理方法 | 紙 / Excel / スプレッドシート / 専用ソフト | データソース整備の工数見積もり |

これにより、有料レポートで**顧客のIT環境に合った具体的なアーキテクチャ**を提案できる。高性能PCがあればローカルLLM、Google Workspaceを使っていればGoogleフォト→スプレッドシート連携、Notionを使っていればNotion API連携など。

## ファネル設計の意図

```
無料レポート（0円）  ← リード獲得。原価ゼロ。大量配布可能
    ↓ 特典: 無料MTG
有料レポート（¥5,000） ← リード品質フィルター。広告費として割り切り
    ↓ MTGでクロージング
導入支援（個別見積もり） ← 収益の柱。自動化システムの構築・納品
    ↓ 自然な継続
保守（月額）        ← 継続収益。改善・拡張・トラブル対応
```

- 無料 → 有料レポートのステップで「お金を払ってでも話を聞きたい」層だけを残す
- 有料レポートは導入支援の**プレゼン資料**であり、それ自体で利益を出す必要はない
- 導入支援の受注が取れれば¥5,000は十分に回収できる
- 小規模事業者にとって¥5,000は「軽い気持ちで試せる」価格帯

## 汎用AIとの差別化（無料版でも担保するポイント）

具体的なAIツール名は出さないが、以下で「自分でAIに聞くより価値がある」と感じさせる:

- [ ] 業種特有の課題への言及（業界固有の慣行・繁忙期・顧客特性を踏まえた分析）
- [ ] 削減時間の数値根拠と計算ロジック（「月8時間 × 時給1,500円 = 月12,000円」レベル）
- [ ] 連携サービスとの組み合わせ提案（LINE公式×AI、freee×自動化、Googleカレンダー×リマインド）
- [ ] 補助金の名称と概要（日本の中小企業向け）
- [ ] 有料レポートで得られる具体的な分析内容の提示
