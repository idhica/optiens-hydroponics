# AI活用支援 — 業務フロー定義

## 目的
顧客接点ごとの対応手順を定義し、属人化を防ぎ、再現可能な営業・納品オペレーションを構築する。
本書は会社資産として運用し、改訂時はバージョンを上げる。

v1.2では、4エージェント並列レビュー（review-agent / strategy-agent / business-analyst / critical-analyst）の指摘39件と、CEO決定（プラン体系再編・出力フォーマットGoogle Slides化・CRM=Supabase Edge Functions＋pg_cron・「相殺」→「充当」）を統合し、「ハッピーパス」一辺倒から「異常系・段階的運用条件」を併載する構造に転換した。

## 戦略目標

- **Pattern B/C比率50%以上を維持する**。Pattern Aは入口・教育目的、収益柱はB/C。
- Pattern A単体では時給換算で月50万報酬を支えきれない（business-analyst試算）。Aの存在意義は「リード品質フィルター・教育・実績アピール材料」。
- Phase A1（〜6ヶ月）目標: 有償案件1件
- Phase A2（7-12ヶ月目）目標: 月1-2件受注、月間売上15-30万円
- Phase A3（2年目）目標: 月3-5件、月間売上50-100万円
- Phase A4（3年目〜）目標: 月間利益40-100万円

### 重要KPI閾値

| KPI | 推奨閾値 | 用途 |
|---|---|---|
| 無料→¥5,000転換率 | 15%以上（3ヶ月平均） | フォーム改善・無料版差別化の見直し閾値 |
| ¥5,000→導入支援転換率 | 20%以上 | レポート品質・MTGクロージングの見直し閾値 |
| ¥30,000要件定義不成約率 | 50%以下 | 提案精度の見直し閾値 |
| 検収後3ヶ月無償サポート | 累計10h上限 | 超過時は保守契約必須化 |
| 保守ライト超過率 | 20%以下 | 超過頻発時はスタンダード推奨 |
| 導入支援最低受注額 | 40万円 | これ未満は¥5,000レポートに誘導 |
| 同時案件並走上限 | 暫定3件 | キャパ崩壊防止（Phase A1実績で改定） |
| スポット相談月受付上限 | 5件/月 | キャパ保護 |
| 応答SLA | 3営業日以内 | ワンオペ前提・常時オン否定 |
| 賠償上限 | 契約金額の1倍 | 業界相場下限・契約書明記 |

## Definition of Ready（DoR）— 全Pattern稼働の前提

DoR（Definition of Ready）= 「v1.2を本番運用扱いに格上げするための必須前提」。
DoR充足前は **Phase A1暫定運用**（口頭合意ベース）として案件を進めることをCEO判断で許容するが、契約書テンプレート完成を最優先する。

### DoRチェックリスト

- [ ] 要件定義業務委託契約書テンプレート完成（弁護士確認済）
- [ ] AI導入支援業務委託契約書テンプレート完成（弁護士確認済）
- [ ] 紹介者契約書テンプレート完成（Pattern C用）
- [ ] 特商法表記サイト掲載済（B2B限定の旨）
- [ ] プライバシーポリシー掲載済
- [ ] 充当時の消費税処理を税理士確認済
- [ ] 不成約時の成果物（要件定義レポート）の著作権帰属確定
- [ ] 全成果物に「Optiens独占的設計に基づく・第三者への開示転用禁止」明記
- [ ] 全自動メールに配信解除リンク付与（特定電子メール法対応）
- [ ] フォームDoS対策（reCAPTCHA・レート制限・手動承認段階）実装済

### DoR未充足時の暫定運用

- 全Pattern暫定運用OK（CEO決定）。ただし「Phase A1暫定運用」と業務メモに明記し、契約書テンプレート完成を最優先タスクとする。
- 暫定運用中の案件は、契約書完成後に**遡って正式契約書で再締結する**ことを顧客と合意しておく。

## Pre-Funnel判定ルール（流入経路別の自動振り分け）

cold MTGリクエスト・直接問い合わせ・各種フォーム入力に対し、自動でA/B/C/C-softに振り分ける。
**cold MTGリクエストはfunnel強制誘導**（自動返信メールでA/B/Cプラン提示）。

### 判定フロー

```
[流入]
 │
 ├─ サイト「無料AI診断」フォーム送信
 │    → Pattern A（標準フロー・無料診断スタート）
 │
 ├─ サイト「詳細レポート申込」フォーム送信
 │    → Pattern A の途中（ステップ4以降）に合流
 │
 ├─ MTG予約フォームから直接予約（cold）
 │    → 自動返信メールでA/B/Cプラン提示し誘導（funnel強制誘導）
 │    → 予約は受けない（Pattern Aの無料診断 or Pattern Bの¥30,000要件定義 or Pattern Cの紹介経由を選んでもらう）
 │
 ├─ メール・電話で「直接導入支援を頼みたい」
 │    → Pattern B（¥30,000有償要件定義経由）
 │
 ├─ 商工会・既存顧客・営業パートナー経由の紹介
 │    │
 │    ├─ 紹介者がOptiensを名指し → Pattern C 標準（無料キックオフMTG継続）
 │    └─ 紹介者が「AIに詳しい会社を紹介」レベル（温度感低い） → Pattern C-soft → Pattern A合流
 │
 └─ 完了後の再相談
      ├─ 完了後3ヶ月超 → 別件無料1件（30-60分・スクリーニング目的）
      └─ 完了後3ヶ月以内 → ¥18,000/90分スポット相談
```

### cold MTGリクエスト自動返信テンプレ（要約）

```
ご連絡ありがとうございます。Optiensでは現在、以下の3プランでご相談を承っております。

A. 無料AI診断レポート（¥0）
   - フォーム入力後、Google Slides形式でレポートをお送りします
   - MTGは含まれません

B. 詳細レポート + MTG（¥5,000・税別）
   - 詳細フォーム入力後、5-8pレポート + 60分MTGをお届けします

C. 有償要件定義（¥30,000・税別）
   - 直接導入支援をご希望の方向け
   - 成約時に初期費用へ全額充当します

詳細は[無料診断ページ]からお進みください。
```

## 全体像

```
[流入チャネル]
 ├─ サイト「無料AI診断」 → Pattern A
 ├─ サイト「詳細レポート申込」 → Pattern A 途中合流
 ├─ MTG予約直接（cold） → funnel強制誘導
 ├─ 「直接導入支援希望」メール・電話 → Pattern B
 ├─ 商工会・紹介・既存顧客 → Pattern C / Pattern C-soft
 └─ 完了後再相談 → 完了後再相談メニュー

         ↓

[Pre-Funnel判定] ← Pattern A / B / C / C-soft / 再相談 のどれかに振り分け

         ↓

Pattern A:      標準フロー（無料診断 → ¥5,000 + 60分MTG → 導入支援 → 保守）
Pattern B:      ¥30,000有償要件定義経由（直接導入支援希望）→ 導入支援 → 保守
Pattern C:      紹介・指名（無料キックオフMTG）→ 導入支援 → 保守
Pattern C-soft: 温度感低い紹介 → Pattern Aに合流
完了後再相談:    3ヶ月超=無料1件 / 3ヶ月以内=¥18,000スポット

         ↓

[共通] 導入支援 → 検収（みなし条項あり）→ 保守 / アップセル
```

### 無料レポートのルール（v1.2で更新）

無料レポートは「業務TOP3 / 仕組みの記述 / 導入支援見積の有無」の3項目に絞った差別化フレームを採用する。詳細は `無料版vs有償版_定義.md` の「無料版との差別化（3項目に絞り込み）」を参照。

社外発信での禁止用語（researcher調査により造語と確定）:
- 「地方創生3.0」（公式用語は地方創生2.0。3.0は造語）
- 「プロクローン」（外部記事の要約・原典特定不能・業界一般用語ではない）

API価格の表現ルール（researcher / business-analyst指摘反映）:
- モデル名・処理規模・前提条件をセットで提示する
- 「月額¥500程度」のような単一値表現は禁止
- 表現例: 「Haiku 4.5・小型モデル+短プロンプト+低頻度なら月数百円。Sonnet/Opusの本格運用は月数千円〜数万円」

### 判定ルール

- ステップ7のMTG後、導入支援に進まない場合 → 失注扱い。CRM（Supabase）に記録、3ヶ月後に再アプローチメール（自動）
- ¥5,000は売上計上、不成約時も返金不要
- 詳細フォームで「MTG不要」がチェックされた場合は文書のみで判断（オプトアウト型）

## Pattern B: ¥30,000有償要件定義フロー（v1.2で全面再構築）

### 適用条件
- 顧客が「詳細レポートは不要、導入支援を直接スタートしたい」と明示した
- すでにAI活用の方向性を自分で固めている層

### v1.2変更点（CEO決定）

- 旧「無料ヒアリング（60-90分）」フローを **廃止**
- 「直接導入支援希望」の顧客は **¥30,000有償要件定義経由** で受ける
- 旧「1顧客1件・3ヶ月クールダウン」判定ルールも廃止（¥30,000フィルタが代替）
- escape hatch（緊急時別ルート）は残す（CEO単独判断で例外的に無料相談を受ける場合は議事録に「escape hatch適用」と明記）

### 根拠
- ¥30,000の本質は「直接導入支援希望者向けの本気度フィルター + 要件定義工数の対価」。法的には準委任契約として、業務完了時に報酬請求権が確定する（不成約時も返金不要）
- 成約時は本契約の初期費用に充当（旧表記「成約時相殺」を「初期費用に充当」に統一・ip-agent法務調査結果反映）

### ステップ

| # | フェーズ | アクション | 担当 | 納品物 / 出力 | 所要 | 自動化候補 |
|---|---|---|---|---|---|---|
| 1 | 意思確認 | 直接導入支援希望をメール/口頭で受領 | CEO | 確認メール | 即時 | 低 |
| 2 | プラン提示 | ¥30,000有償要件定義を提示（成約時に初期費用へ充当） | CEO | 案内メール | 即時 | 低 |
| 3 | 要件定義契約 | 要件定義業務委託契約書締結 + ¥30,000請求 | CEO | 契約書 + 請求書 | 1-3営業日 | 中（freee MCP） |
| 4 | 簡易事前フォーム | 詳細フォーム（IT環境表+ファイル添付欄。Pattern Aと同一フォーム） | システム | 入力依頼メール | 即時 | 高（実装済） |

| 8 | 提案MTG | 提案説明 + 質疑 + クロージング（**修正対応1回まで含む**） | CEO | MTG議事メモ + 受発注合意 | 30-60分 | 低 |
| 9 | 本契約締結 | AI導入支援業務委託契約書締結 + **着手金50%請求（¥30,000充当）** | CEO | 契約書 + 請求書 | 1-3営業日 | 中（freee MCP） |
| → | | **以下「共通：導入支援フロー」へ合流** | | | | |

### 工数上限（CEO決定）

- ヒアリング: 90分以内
- 提案書: 3-5pまで
- 修正対応: 1回まで
- ヒアリング後10営業日以内に提案書提出

### 充当条項（契約書テンプレート必須）

ip-agent法務調査§2-2準拠。契約書には必ず以下の条項を含める:

```
第X条（要件定義費用の充当）
1. 甲（顧客）は乙（Optiens）に対し、要件定義業務の対価として金30,000円
   （消費税別）を契約締結時に支払う。
2. 本要件定義業務終了後、甲乙間で別途AI導入支援業務に関する本契約が
   締結された場合、前項の費用は本契約の初期費用の一部に充当する。
3. 前項の本契約が締結されなかった場合、第1項の費用は返金しない。
   本要件定義業務は準委任契約として、業務完了時に報酬請求権が確定する。
```

### 判定ルール

- 失注時は売上計上、¥30,000は返金不要
- 受注時は¥30,000を本契約の初期費用に充当（消費税処理は税理士確認待ち）
- escape hatch適用時は議事録に明記し、月次レビューで件数を集計
- 同一顧客の再依頼で要件定義が完全に作り直しになる場合は、再度¥30,000を請求

## Pattern C: 直行フロー（紹介・指名）

### 適用条件
- 商工会・既存顧客・営業パートナー経由の紹介
- 顧客が「Optiensに頼みたい」と指名で来ている
- すでに信頼ベースが構築されている

### Pattern C-soft（v1.2で新設・内部分割）

商工会経由の温度感が低い紹介を **Pattern C-soft** として内部分割。
- 紹介者が「Optiensを名指し」→ Pattern C 標準
- 紹介者が「AIに詳しい会社を紹介」レベル → Pattern C-soft → Pattern A合流（無料診断レポート提示から始める）
