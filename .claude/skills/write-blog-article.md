# ブログ記事自動生成スキル

このスキルはOptiens公式ブログの記事を自動生成し、デプロイまで行います。

## 実行手順

### Step 1: テーマ選定
以下の6カテゴリからローテーションでテーマを選定する。前回の記事カテゴリと重複しない。
既存記事で最も少ないカテゴリを優先的に選ぶ（カテゴリバランスを維持）。

| カテゴリ | 目的 | テーマ例 | ターゲット読者 |
|---|---|---|---|
| agriculture | 水耕栽培の魅力・始め方を伝え、新規参入者を増やす | 品目別栽培ガイド、収益シミュレーション、失敗事例と対策、季節Tips | 水耕栽培を始めたい人・検討者 |
| business | 水耕栽培ビジネスの経営情報を提供 | 補助金活用、販路開拓、ユニットエコノミクス、法人設立体験記、コスト管理 | 事業者・経営者・就労支援事業所の管理者 |
| technology | IoT×水耕栽培の技術力をアピール | IoTセンサー構築、Raspberry Pi活用、Zigbee/MQTT、LED制御、自動化 | 技術導入を検討する事業者・エンジニア |
| welfare | 農福連携・就労支援での水耕栽培活用 | 就労支援事業所での導入メリット、作業設計、工賃向上、農福連携事例 | 就労支援事業所・自治体（Phase 5の直接ターゲット） |
| AI | AI×農業の最前線 | Claude Code × MCP、農業AI動向、画像認識による病害検知 | 技術好き・メディア露出 |
| announcement | 事業進捗 | マイルストーン達成、新品目追加、パートナーシップ | 既存フォロワー |

### カテゴリ戦略（Phase 5を見据えたコンテンツマーケティング）
ブログの目的は「水耕栽培の魅力を発信→新規事業者を増やす→Optiensの技術・機材導入の仕事を受注」。
- agriculture/business/welfare: 「水耕栽培を始めたい・拡大したい人」を育てる記事群（集客の入口）
- technology: Optiensの技術力を示し、導入検討者の信頼を獲得する記事群
- AI: 先進性アピール・メディア露出狙い
- announcement: 事業の実在感・進捗を伝える

### ローテーションルール
週3回（月・水・金）投稿。以下のローテーションを基本とし、カテゴリバランスを最優先に調整:
- **月曜**: agriculture / business / welfare の中で最も少ないもの
- **水曜**: technology / AI の中で最も少ないもの
- **金曜**: 全カテゴリの中で最も少ないもの（announcement含む）

テーマ選定時は以下のエージェントを使用:
1. **researcher**エージェントで最新の水耕栽培・農業IoTニュースを調査
2. 既存記事（`src/content/blog/`）と重複しないテーマを選定
3. SEOキーワードを意識（「水耕栽培 バジル」「IoT 農業」「北杜市 ハーブ」等）

### Step 2: 記事執筆
以下の構成で2000-3000字の記事を作成:

```
---
title: '記事タイトル（SEOを意識、30-60字）'
date: 'YYYY-MM-DD'（当日の日付）
category: 'agriculture' | 'AI' | 'technology' | 'social' | 'announcement'
excerpt: '要約（100-150字）'
image: '/images/blog/スラッグ名.webp'
---

## 導入（なぜこのテーマが重要か）
## 本論（2-3セクション、データ・事例を含む）
## Optiensの取り組み（自社の関連性を自然に言及）
## まとめ
```

### Step 3: ファクトチェック（最重要）

**3a. プロジェクト進捗との整合性チェック**
1. `src/content/blog/_project-status.md` を必ず読み、現在のフェーズ・完了済み項目・未完了項目を把握する
2. 記事内のOptiensに関する記述が進捗状況と矛盾していないか検証する
3. 未完了の項目を完了済みかのように記述していないか確認する
4. 禁止表現リストに該当する表現がないか確認する

**3b. その他のファクトチェック**
- **統計データ**: 出典URLが有効か確認（WebFetchで検証）
- **技術的記述**: 正確性を確認（スペック値、プロトコル名等）
- **禁止事項**: CLAUDE.mdの禁止表現が含まれていないか確認
- **日付・数値**: 矛盾がないか確認

### Step 4: アイキャッチ画像生成
`scripts/generate-blog.py` を使用してアイキャッチ画像を生成:

```bash
python scripts/generate-blog.py "記事テーマに合わせた英語プロンプト" "public/images/blog/スラッグ名.webp"
```

内部でImagen 4.0（PNG生成→WebP変換）を使用。GEMINI_API_KEYは.envから自動読み込み。

画像プロンプトのルール:
- Optiensのブランドカラー（#2e574c, #5ea89a）を意識
- テキストは含めない（画像のみ）
- モダン・クリーン・プロフェッショナルなスタイル
- 水耕栽培・IoT・ハーブ・テクノロジーのビジュアル

### Step 5: ファイル保存
- 記事: `src/content/blog/{slug}.md`
- 画像: `public/images/blog/{slug}.webp`

### Step 6: ビルド確認
```bash
npx astro build
```
エラーがないことを確認。

### Step 7: Git commit & push
```bash
git add src/content/blog/{slug}.md public/images/blog/{slug}.webp
git commit -m "blog: {記事タイトルの要約}"
git push origin main
```

## 注意事項
- CLAUDE.mdの禁止事項を必ず遵守
- 他サイトの文章をコピーしない（参考にしてオリジナルで書く）
- 出典は記事末尾にまとめる
- 画像生成に失敗した場合は画像なしで記事を公開（imageフィールドを省略）

## 【必須】事実確認ルール（フェーズ整合性）

Optiensは現在 **Phase 1（自宅テスト栽培）の開始前段階** です。
以下のルールを必ず守ってください。

### 禁止表現（事実に反する）
- 「センサーデータを収集しています」「モニタリングしています」など、システムが現在稼働中であるかのような現在形
- 「〇〇件のデータが蓄積されています」など、具体的な実績数値を記載すること（架空データ禁止）
- 「バジルを収穫しました」「〇〇gの収穫ができました」など、栽培・収穫の実績を記載すること
- 「レストランに納品しています」など、販売実績を記載すること

### 許可される表現（設計・計画中）
- 「〇〇する構成を設計しています」「〇〇する予定です」「構築中です」
- 「〇〇を目標にしています」「〇〇を想定しています」
- 技術解説記事として「こうすれば実現できる」という一般論・実装方法の解説

### 判断基準
「Optiensでの取り組み」「Optiensでの実運用」などの自社紹介セクションで
現在進行形を使う場合は、「現在Phase 1のシステム構築を進めています」程度に留める。
実績データ（収集件数・収穫量・売上）は一切記載しない。
