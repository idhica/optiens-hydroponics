# 無料AI診断レポート生成スキル

フォームから受信した企業情報をもとに、無料AI診断レポートを生成するスキル。
CEOがメール通知を受け取った後、このスキルを呼び出してレポートを作成する。

## 起動方法

```
/generate-diagnosis-report [企業名]
```

引数なしの場合はSupabaseの `diagnosis_leads` テーブルから `status = 'new'` の最新1件を取得する。

## 実行手順

### Step 1: 申込データの取得

Supabaseの `diagnosis_leads` テーブルから対象企業のデータを取得する。
取得するフィールド:
- company_name, person_name, email, industry, employee_count
- ai_level, challenges, interests
- business_description, daily_tasks, current_tools, free_text

### Step 2: 業種別ナレッジの参照

`src/content/blog/` および `src/pages/ai-consulting/sample-*.astro` から、
該当業種に近いサンプルレポートを読み込み、具体的な事例・提案パターンを把握する。

業種マッピング:
| industry値 | 参照するサンプルレポート |
|---|---|
| accommodation | sample-pension |
| restaurant | sample-restaurant |
| construction | sample-construction |
| winery | sample-winery |
| outdoor | sample-outdoor |
| bakery | sample-bakery |
| agriculture | sample-farmer |
| municipality | sample-municipality |
| その他 | 最も近い業種のサンプル + 汎用パターン |

### Step 3: レポート生成

以下のテンプレート構成に従ってレポートを生成する。

## 差別化チェックリスト（生成時に必ず確認）

汎用AIが出す一般論との差別化を担保するため、以下をチェック:

- [ ] **AIツール名を出していないか**: Claude/ChatGPT/Gemini等の具体名は有償版の差別化ポイント。無料版では「AIサービス」「AIチャット」と一般化する
- [ ] **AI以外の連携サービス名は出しているか**: LINE公式、Googleカレンダー、freee、Notion等は記載OK（AIツールではないため）
- [ ] **コストレンジが入っているか**: 「数千円程度」レベル。具体的なツール名・正確な価格は有償版で
- [ ] **削減時間の数値根拠があるか**: 「効率化できます」ではなく「月8時間削減」「年間96時間」
- [ ] **業種特有の文脈があるか**: サンプルレポートから得た業界固有の知識を反映
- [ ] **使用中ツールとの連携を提案しているか**: current_toolsに記載のツールとAIの組み合わせ
- [ ] **「自力で試せること」と「伴走が必要なこと」を分けているか**: アップセルへの自然な導線
- [ ] **補助金情報があるか**: 持続化補助金のみ記載（デジタル化補助金はIT支援事業者登録が必要なため不可）
- [ ] **セキュリティの具体的回答があるか**: 特にchallengesで選択されている場合
- [ ] **有償版の広告セクション（第7章）があるか**: 有償版の成果物一覧・差別化ポイント・補助金活用を明示
- [ ] **PDF納品を前提とした体裁か**: Markdown→PDF変換時にきれいに見えるか

## 汎用AIとの差別化ポイント（なぜ自分でやるより価値があるか）

無料版でAIツール名を出さない代わりに、以下で「自分でAIに聞くより価値がある」と感じさせる:

1. **業種特有の課題分析**: 業界固有の慣行・繁忙期・顧客特性を踏まえた分析（汎用AIは一般論）
2. **日本の中小企業文脈**: 補助金、確定申告、freee/弥生連携、LINE公式活用等（汎用AIは米国中心）
3. **具体的な数値根拠**: 「月○時間削減」「年間○円相当」レベルの試算（汎用AIは概念止まり）
4. **連携サービスの組み合わせ提案**: LINE公式×AI、Googleカレンダー×リマインド等（汎用AIは単体提案）
5. **セキュリティの日本語ファクト**: 総務省ガイドライン、IPA資料への参照（汎用AIは英語圏中心）
6. **有償版への具体的な導線**: 有償版で得られる成果物を明示し、次のアクションを促す
7. **監修の信頼性**: 「AIが出力→人間が監修」の二段階プロセスで品質保証

### Step 4: レポートの保存

生成したレポートを以下に保存する:
```
executive/ai-consulting/reports/{company_name}_{date}.md
```

### Step 4.5: PDF生成

Markdown → PDF変換を実行する:
```bash
node scripts/generate-report-pdf.mjs executive/ai-consulting/reports/{company_name}_{date}.md
```
- 同じディレクトリに `{company_name}_{date}.pdf` が生成される
- CSSテンプレート: `executive/ai-consulting/pdf-template.css`（Optiensブランドカラー適用済み）
- CEOがレビュー後、このPDFをメールで送付する

### Step 5: ステータス更新

Supabaseの `diagnosis_leads` テーブルで該当レコードの `status` を更新する。

```
MCP Supabase または curl で実行:
UPDATE diagnosis_leads SET status = 'report_created', updated_at = now() WHERE id = {id};
```

#### ステータスフロー

| status | 意味 | 遷移タイミング |
|---|---|---|
| `new` | 申込受付済み（未着手） | フォーム送信時（API自動） |
| `report_created` | レポート作成済み（未送付） | 本スキルのStep 5 |
| `sent` | レポート送付済み | CEOがメール送付後に手動更新 |
| `converted` | 有償版に転換 | 有償契約成立時に手動更新 |

**確認方法**: `status = 'new'` のレコードが未対応の申込。
```sql
SELECT id, company_name, industry, created_at FROM diagnosis_leads WHERE status = 'new' ORDER BY created_at DESC;
```

### Step 6: CEO への報告

レポートのサマリー（TOP3の提案と主要な差別化ポイント）をCEOに報告する。
CEOがレビュー・編集した後、メールで送付し、ステータスを `'sent'` に更新する。

## 無料版と有償版の位置づけ

本スキルで生成するのは **無料版（簡易版）** のレポート。
有償版（15万円）との差分は `executive/ai-consulting/無料版vs有償版_定義.md` を参照。

| | 無料版（本スキルで生成） | 有償版（15万円） |
|---|---|---|
| 情報源 | Webフォーム入力のみ | 現地/オンラインヒアリング 2-3h |
| 提案数 | TOP3 | 5-7件 + プロンプト例・生成例付き |
| ROI | コスト試算（概算） | 詳細ROI + 売上間接インパクト |
| ロードマップ | 次のステップ（概要） | 月次実行計画 3-6ヶ月分 |
| 導入支援 | なし（レポートのみ） | 初期セットアップ 1-2h 含む |
| 納期 | 1-2営業日 | 2-3週間 |

## 注意事項

- レポートの最終送付はCEOが手動で行う（Phase 1）
- ツールの価格情報は最新のものを使用する。不明な場合は「要確認」と記載
- 補助金情報は公募中のもののみ記載。終了したものは記載しない
- 個人情報（メールアドレス等）をレポート本文に記載しない
